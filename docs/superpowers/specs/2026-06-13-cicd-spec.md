## New Workflow: `ci.yml`

**Trigger:** `pull_request` (any branch â†’ main) and `push` (to `main`)

### Job 1: `test-backend`

```yaml
runs-on: ubuntu-latest
services:
  postgres:
    image: postgres:16
    env:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: tenant_db_test
    ports: ["5432:5432"]
    options: >-
      --health-cmd pg_isready
      --health-interval 10s
      --health-timeout 5s
      --health-retries 5
```

Steps:
1. `actions/checkout@v4`
2. `oven-sh/setup-bun@v2` with `bun-version: 1.1.29`
3. `bun install` in `./backend`
4. Push schema to test DB: `PG_DB=tenant_db_test bunx drizzle-kit push --force` in `./backend`
5. Run tests: `bun test` in `./backend`

Env vars for all steps:
```
PG_HOST: localhost
PG_PORT: 5432
PG_USER: postgres
PG_PASSWORD: postgres
```

### Job 2: `test-frontend`

```yaml
runs-on: ubuntu-latest
```

Steps:
1. `actions/checkout@v4`
2. `oven-sh/setup-bun@v2` with `bun-version: 1.1.29`
3. `bun install` in `./frontend`
4. Run tests: `bun run test` in `./frontend` (which calls `vitest run`)

No DB service needed.

### Job 3: `build-images` (conditional)

Only runs on `push` to `main` (not on PRs). Depends on both test jobs.

```yaml
needs: [test-backend, test-frontend]
if: github.event_name == 'push' && github.ref == 'refs/heads/main'
```

Steps mirror existing `ghcr-images.yml` build jobs for backend + frontend. Tags with `sha` + `latest`.
